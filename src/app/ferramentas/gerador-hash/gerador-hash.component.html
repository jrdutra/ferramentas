Como gerar o hash de um texto

Basta digitar ou colar o texto no campo acima. Os quatro hashes são calculados automaticamente em tempo real, sem necessidade de clicar em nenhum botão. Use o ícone de cópia ao lado de cada resultado para copiar o valor para a área de transferência.

Alternar entre maiúsculas e minúsculas

Por padrão os hashes são exibidos em letras minúsculas. Clique no botão de alternância no canto superior direito do painel para exibir os valores em letras maiúsculas, que é o formato exigido por algumas APIs e sistemas legados.

O que é uma Função de Hash Criptográfico

Uma função de hash criptográfico é um algoritmo matemático que transforma qualquer quantidade de dados em uma sequência de tamanho fixo, chamada de hash ou digest. A mesma entrada sempre produz o mesmo hash, mas qualquer alteração mínima na entrada — mesmo um único caractere — resulta em um hash completamente diferente. Essa propriedade é conhecida como efeito avalanche.

Funções de hash são amplamente usadas para verificar a integridade de arquivos, armazenar senhas de forma segura, assinar digitalmente documentos e garantir a autenticidade de mensagens. É importante diferenciar hash de criptografia: o hash é uma operação unidirecional, ou seja, não é possível recuperar o texto original a partir do hash.

MD5

Criado em 1991 por Ronald Rivest, o MD5 produz um hash de 128 bits, representado por 32 caracteres hexadecimais. Apesar de ainda ser usado para verificação de integridade de arquivos, não é recomendado para fins de segurança, pois vulnerabilidades conhecidas permitem criar colisões — entradas diferentes que geram o mesmo hash.

SHA-1

Desenvolvido pela NSA e publicado em 1995, o SHA-1 produz um hash de 160 bits (40 caracteres hex). Foi o padrão dominante por anos em certificados SSL e assinaturas digitais, mas em 2017 pesquisadores demonstraram a primeira colisão prática, tornando-o obsoleto para uso criptográfico. Ainda é encontrado em sistemas legados.

SHA-256 e SHA-512

Ambos pertencem à família SHA-2, publicada pela NSA em 2001. O SHA-256 produz 256 bits (64 caracteres hex) e é o padrão recomendado para a maioria das aplicações de segurança, sendo usado em certificados TLS, Bitcoin e assinaturas de código. O SHA-512 produz 512 bits (128 caracteres hex) e oferece maior margem de segurança, sendo preferido em contextos que exigem resistência extra a ataques de força bruta.

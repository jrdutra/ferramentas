Instruções

Como comparar dois JSONs

Cole o JSON original no campo JSON A e o JSON modificado no campo JSON B, depois clique em Comparar. A ferramenta percorre recursivamente a estrutura dos dois objetos e exibe as diferenças lado a lado, campo a campo.

Formatar o JSON antes de comparar

Caso o JSON esteja minificado ou stringficado, use o botão auto_fix_high em cada campo para formatá-lo antes de comparar. Isso facilita a leitura e garante que diferenças de formatação não sejam interpretadas como mudanças de conteúdo.

Interpretando o resultado

Campos em verde existem apenas no JSON B — foram adicionados. Campos em vermelho existem apenas no JSON A — foram removidos. Campos em amarelo existem nos dois JSONs, mas com valores diferentes — foram modificados. Campos sem destaque são idênticos nos dois lados.

O que é JSON

JSON (JavaScript Object Notation) é um formato leve de troca de dados, fácil de ler por humanos e simples de interpretar por máquinas. Ele é baseado em um subconjunto da linguagem JavaScript, mas é completamente independente de linguagem. JSON é amplamente utilizado para transmitir dados estruturados entre servidores e clientes na web, bem como em arquivos de configuração, APIs REST e armazenamento de dados.

Foi formalizado por Douglas Crockford na especificação RFC 4627 em 2006 e, desde então, tornou-se um dos formatos de dados mais adotados no desenvolvimento de software. Um objeto JSON é composto por pares chave/valor, onde os valores podem ser strings, números, booleanos, arrays, outros objetos ou null — o que o torna extremamente flexível para representar praticamente qualquer estrutura de dados.
